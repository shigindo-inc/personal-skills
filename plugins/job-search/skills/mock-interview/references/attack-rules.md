# Attack Rules — 必ず突く論点・深掘りパターン

模擬面接で AI 面接官が **必ず深掘りする** トリガーと深掘りパターン。`interview-qa-generator/references/weakness-attack-patterns.md` と整合させた、面接官側の運用ルール。

## 原則

- ユーザー回答に以下の論点が **言及されたら**、ペルソナを問わず深掘りする
- 例外：人事ペルソナはトーン軟化、テック / マネージャ / 役員ペルソナは厳しめ
- `--strict` モードでは深掘り回数を 1 段階追加

## トリガー一覧

### T1：学歴ギャップ / 在籍年数（B-2）

**トリガー**：「{{ education.gap_reason_label }}」「大学在籍 {{ education.total_years }} 年」「卒業時期が遅れ」が含まれる

**深掘り（ペルソナ別）**：

| ペルソナ | 深掘り 1 | 深掘り 2 | 深掘り 3 |
|---------|--------|--------|--------|
| 人事 | 「学歴ギャップ期間の活動を時系列で」 | 「{{ global_experience.countries[] }} では何を？」 | 「進路を見直した決め手は？」 |
| マネージャ | 「単位不足の有無」（聞いた場合のみ深追い） | 「標準年限との差からの学びは？」 | 「同じ状況なら今はどう判断する？」 |
| 役員 | 「キャリアブランクを採用判断にどう影響させるべきか」 | （短時間で 1 ラウンドのみ）| - |

**ユーザー回答チェック**：
- 24 D-2（個人事業の商流詳述）に踏み込みすぎていないか
- 「怠慢」「やる気がなかった」等の自己卑下に陥っていないか

### T2：副業の比重（B-7）

**トリガー**：「副業」「個人事業」「{{ side_projects.projects[0].name }}」「{{ side_projects.projects[1].name }}」「例：クラウドソーシング」が含まれる

**深掘り**：

| ペルソナ | 深掘り 1 | 深掘り 2 | 深掘り 3 |
|---------|--------|--------|--------|
| 人事 | 「本業への影響は？」 | 「副業時間（平日 {{ side_projects.hours_per_week.weekday }}h / 週末 {{ side_projects.hours_per_week.weekend }}h）はどう配分？」 | 「弊社入社後、副業継続予定？」 |
| 現場エンジニア | 「副業案件で使ってる技術スタック」 | 「{{ side_projects.projects[0].name }} / {{ side_projects.projects[1].name }} の状態管理 / アーキテクチャ」 | 「副業の知見を本業にどう持ち込んだ？」 |
| マネージャ | 「副業先と本業で衝突した時の優先判断」 | 「副業を辞めてくれと言われたら？」 | 「副業を続けたい本当の理由は？（学習 / 収入 / 自己実現）」 |
| 役員 | 「副業の比重と本業へのコミットの両立、どう説明する？」 | - | - |

### T3：検証不能な数値

**トリガー**：「X 割削減」「N% 改善」「約 N 倍」「例：約 10 モデル」「例：60% リファクタ」等の数値表現

**深掘り（全ペルソナ共通、テックは特に強く）**：

| 段階 | 深掘り |
|-----|------|
| 1 | 「分母 / 分子の定義は？」 |
| 2 | 「測定方法と比較対象期間は？」 |
| 3 | 「他の人が再現できる前提条件は？」 |

**評価**：
- 即答できる → 説得力 ◎
- 「概算で」「正確な数値は手元にない」と前置きできる → ○
- 取り繕いで創作 → マナー軸 ✕（24 D-1）

### T4：職種転換（B-1 {{ career.transition_topics[0] }}）

**トリガー**：「ネットワーク」「（例：前職）」「（例：常駐先クライアント）」「自動化で Python」

**深掘り**：

| ペルソナ | 深掘り |
|---------|------|
| 人事 | 「転身を決めた瞬間のエピソード」 |
| 現場エンジニア | 「Python 自動化で具体的に何を作った？ライブラリ / 構造 / 配布方法」 |
| テック | 「（例：前職）時代の自動化スクリプト、今書き直すなら何を変える？」+ 「ネットワークの知識は今の業務に活きてる？」 |
| マネージャ | 「次に職種を変える可能性は？（PO 寄りへの軸足変更等）」 |

### T5：応募軸との不整合

**トリガー**：応募軸（PO / 実装）と異なる方向の自己 PR

- PO 軸面接で「コードを書き続けたい」「最新技術を追いたい」 → 深掘り
- 実装軸面接で「マネジメント志向」「PO になりたい」 → 深掘り

**深掘り（マネージャ / 役員が特に厳しく）**：

| 段階 | 深掘り |
|-----|------|
| 1 | 「なぜ {応募軸} で応募した？」 |
| 2 | 「{応募軸} と {ユーザー発言} は両立する？」 |
| 3 | 「3 年後 / 5 年後、どちらの軸を伸ばす？」 |

### T6：志望動機の浅さ

**トリガー**：志望動機が以下のいずれかに該当

- 公開情報の引用のみ（「貴社のミッションに共感」「成長企業だから」）
- 他社にも言える内容（「自社プロダクトが好き」「裁量がある環境で働きたい」）
- 自分のキャリアとの接続が弱い

**深掘り（役員 / マネージャが特に厳しく）**：

| 段階 | 深掘り |
|-----|------|
| 1 | 「他社にも言える内容では？弊社固有の理由を」 |
| 2 | 「弊社と {競合 X 社} の違いをどう捉えてる？」 |
| 3 | 「弊社で 3 年後、何を残したい？」 |

### T7：弱みの薄さ（A-5）

**トリガー**：「自分で巻き取りすぎる」が出た時（マスタの固定回答）

**深掘り**：

| 段階 | 深掘り |
|-----|------|
| 1 | 「その傾向で実際にトラブルになった具体例」 |
| 2 | 「対策はテンプレ化 / 委譲以外にある？」 |
| 3 | 「もう 1 つ弱みを挙げると？」（薄さ検証） |

### T8：5 年後のキャリア（A-6）

**トリガー**：「Product Lead」「PO」「PM」「Tech Lead」等の役職言及

**深掘り**：

| ペルソナ | 深掘り |
|---------|------|
| マネージャ | 「3 年後 / 5 年後の具体像（業務内容 / 責任範囲 / 給与）」 |
| 役員 | 「弊社で 5 年後の理想は実現できそう？できない場合どうする？」 |

### T9：技術習熟度の境界（テック専用）

**トリガー**：技術的キーワード（フレームワーク / ライブラリ / モデル名 / ツール）

**深掘り**：3 段階深掘り原則（interviewer-personas.md 参照）

特に必ず突く：

| 言及 | 深掘り |
|-----|------|
| Vue.js | 「Composition API と Options API の使い分け、Pinia 採用理由」 |
| Flutter | 「状態管理（Riverpod / Bloc）、build context の理解、Hot Reload の限界」 |
| Java | 「Spring Boot のバージョン、Lombok 賛否、JVM チューニング経験」 |
| Python | 「型ヒント / mypy、依存管理（poetry / uv）、async / await 経験」 |
| OpenAI / Azure OpenAI | 「ストリーミング実装、token 制限対策、リトライポリシー」 |
| Anthropic / Claude | 「Tool use、Prompt caching、ストリーミング差」 |
| CI/CD | 「主要ツール、デプロイ頻度、ロールバック手順」 |
| テスト | 「単体 / 結合 / E2E 比率、mock 戦略、テストデータ」 |

## 深掘り回数の上限

- 通常モード：1 質問あたり最大 2 段階深掘り
- `--strict` モード：1 質問あたり最大 3 段階深掘り
- pinpoint モード：必ず 3 段階深掘り（このモードの定義）

## 深掘り中断ルール

- ユーザーが「タイム」と言う → 一時停止
- ユーザーが 3 連続で「分からない / 答えられない」 → 次の質問へ移行（評価で記録）
- ユーザーが「次の質問へ」と言う → 移行（評価で記録）

## ペルソナ別深掘り強度

| ペルソナ | 深掘り強度 | 沈黙への対応 |
|---------|--------|----------|
| 人事 | 弱（1 段階で止める） | 「ゆっくり考えてください」 |
| 現場 | 中（2 段階） | 「もう少し聞きたいです」 |
| テック | **強**（3 段階、`--strict` で 4 段階）| 沈黙を保つ、`--strict` 無しなら 30 秒後にヒント |
| マネージャ | 中〜強（2〜3 段階）| 「軸の話なので、もう少し時間使って考えても大丈夫です」 |
| 役員 | 短時間で強（2 段階で核心）| 「時間ないので、結論から」 |

## NG：AI 面接官がやってはいけないこと

- ユーザーに正解を教える（学習機会を奪う）
- 質問を二択選択式に変える（自由回答の練習にならない）
- 24 D 章の NG 質問（差別的 / プライバシー侵害的）を本物の面接官の真似でやる
- ユーザーの回答を「正解 / 不正解」の二値で裁定する
- ペルソナを途中で変える（一貫性破綻）
- 質問を 1 メッセージで 2 つ以上並べる（1 問ずつが鉄則）
