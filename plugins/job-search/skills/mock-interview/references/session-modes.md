# Session Modes — 4 モードの質問プール構築 + 進行台本

`--mode` 引数で選択する 4 つのセッションモード。

## モード比較

| モード | 想定時間 | 質問数 | 特徴 | 質問プール |
|-------|--------|------|------|---------|
| full | 30〜60 分 | 5〜10 | 自己紹介 〜 逆質問まで通し | 企業特化（あれば）+ 24 マスタからラウンド向けにフィルタ |
| pinpoint | 5〜15 分 | 1 + 深掘り 3 | 特定 Q（B-2 / B-4 / Q9 等）だけを 3 段階深掘り | `--question` 指定の 1 問 |
| weakness | 15〜30 分 | 3〜5 | 「最近答えられない論点」だけ | `面接ログ/` Grep + `feedback-integrator` 出力 + `47` 追加深掘り |
| cold | 15〜30 分 | 5〜8 | 想定外の角度に強くなる練習 | 24 マスタからランダム、企業特化質問は使わない |

## 1. full モード

### 質問プール構築

1. **自己紹介**（A-1）必須、必ず冒頭 1 問目
2. **転職理由 + 志望動機**（A-2, A-3）必須、2〜3 問目
3. **企業特化質問**（`--company` 指定時、`想定問答_カスタム.md` の Q20〜から 2〜3 問）
4. **応募軸の核心質問**（PO 軸：B-9 / E-3a / Q9、実装軸：C-1 / Q4 / Q6）
5. **経歴深掘り**（B-2 学歴ギャップ、B-7 副業、B-1 職種転換のうちラウンドに応じて 1〜2 問）
6. **条件・働き方**（人事ペルソナ時：Q13〜Q16 のうち 1〜2 問）
7. **逆質問タイム**（最後、ユーザーから 1〜2 件聞く想定。ただし AI が回答するのは概略のみ）

### ラウンド別の質問数 / 深度

| ラウンド | 質問数 | 深掘り頻度 |
|---------|------|---------|
| カジュアル | 5 | 軽め（具体例 1 回） |
| 一次 | 7 | 中程度（具体例 + 1 段深掘り） |
| 二次 | 8 | 強め（具体例 + 2 段深掘り） |
| 最終 | 6 | 短時間で核心を突く（志望動機 + 5 年後を厚く） |

### 進行台本

```
[オープニング]
それでは {ペルソナ} 役の {ラウンド} 面接を始めます。
時間目安：約 {N} 分（{N} 問）

[Q1] まず、自己紹介をお願いします。{ペルソナ追加注文}

[ユーザー回答]

[評価判定（内部）]：
  - 30 秒 / 1 分 / 2 分 のどれで答えたか
  - 強み 3 つを盛り込んだか
  - 副業に言及したか
  → ペルソナのトーンで反応（人事：「ありがとうございます」/ テック：「最後の点、もう少し具体的に……」）

[Q2] {次の質問}
...

[終了直前]
こちらからの質問は以上です。最後に、{ユーザー名} さんからご質問があればお願いします。

[ユーザー逆質問 1〜2 件]
[AI ペルソナとして概略回答（深入りしない）]

それでは、本日はありがとうございました。
評価フェーズに移ります。
```

## 2. pinpoint モード

### 質問プール構築

`--question <章番号>` で指定された 1 問を 3 段階深掘り：

| ラウンド | 進行 |
|---------|------|
| 1 | 主質問を提示、ユーザー回答 |
| 2 | 回答の弱い箇所 / 抽象的な部分を 1 つ突く |
| 3 | さらに突き、「失敗 / 学び / 今ならどう変えるか」で締める |

### 推奨される pinpoint 対象

最も練習価値が高い Q（過去ログで詰まったもの優先）：
- B-2 学歴ギャップ / 在籍年数
- B-4 （例：現職）生成 AI の主担当
- B-7 副業の比重
- B-7a 希望年収
- Q9 プロダクト意思決定（PO 軸時）
- C-2 テスト戦略（テックペルソナ時）

### 進行台本

```
[オープニング]
{Q 番号} の pinpoint 練習を始めます。
このセッションでは 1 つの質問を 3 段階で深掘りします。

[Q1（メイン）]
{質問本文}

[ユーザー回答]

[Q1.1（深掘り 1）]
{弱い点を 1 つ突く}

[ユーザー回答]

[Q1.2（深掘り 2 / 締め）]
{さらに突く + 「失敗経験 / 学び / 今ならどう変えるか」}

[ユーザー回答]

[評価フェーズへ]
```

## 3. weakness モード

### 質問プール構築

過去の MOCK ログ + 本番ログ + `feedback-integrator` 出力から「答えられない論点」を集計：

1. **Grep 対象**：
   - `${COMPANIES_DIR}/*/面接ログ/MOCK_*.md` の `🔧 改善余地` セクション
   - `${COMPANIES_DIR}/*/面接ログ/{本番}.md` の「うまく答えられなかった点」
   - `${BEHAVIOR_DOC}` の「追加で深掘りしたい質問」
2. **頻出論点 5 件をピック**：3 件以上の文書で言及されている論点を優先
3. **各論点に対応する Q を 24 マスタから 1 つずつ選択**
4. **テックペルソナ寄りで深掘り**（弱点克服には強度が必要）

### 進行台本

```
[オープニング]
weakness モード：直近のログから「答えに詰まった論点」を抽出しました。

【今日の練習論点】
1. {論点 A}（出典：{ログファイル}, {ログファイル}）
2. {論点 B}（出典：{ログファイル}）
...

それぞれ 1 問ずつ厳しめに突きます。準備よければ「はい」と返答してください。

[ユーザー：はい]

[Q1] {論点 A に対応する質問}
...
```

## 4. cold モード

### 質問プール構築

24 マスタから **完全ランダムに 5〜8 問**。企業特化質問は使わない（コールド = 知らない企業の面接想定）：

1. A 章から 1〜2 問
2. B 章から 2〜3 問
3. C 章から 1〜2 問
4. ストレス / 対人（Q17〜Q19）から 1 問

意図：想定外の角度に強くなる、暗記回答からの脱却

### 進行台本

```
[オープニング]
cold モード：応募企業を特定しない想定で、ランダムに質問します。
暗記回答ではなく、その場で考える練習です。

[Q1〜Q5（ランダム順）]
...
```

## モード組合せ / 連続実行

- **full → weakness**：本番想定セッション後、見つかった弱点を集中練習
- **pinpoint → full**：苦手 Q を集中練習後、全体に統合
- **cold → 自己評価**：暗記から離れた素の実力測定

ただし **1 セッションは 1 モード固定**。複数モードは別セッションで実施。

## 質問プールから除外するもの

どのモードでも以下は除外：
- E 章（逆質問）：本 skill は「逆質問する役」を AI が演じない（`reverse-questions-curator` の領分）
- D 章（NG ルール）：AI 面接官が NG 質問をしてユーザーに NG 回答を誘発しない

## モード未指定時のデフォルト推定

`--mode` 未指定時の推定ロジック：

| ユーザー発話 / 状況 | 推定モード |
|----------------|---------|
| 「○○社の面接練習」 | full + `--company {名}` |
| 「B-2 の練習」「学歴ギャップの答え方練習」 | pinpoint + `--question B-2` |
| 「最近答えられない論点で」「弱点集中」 | weakness |
| 「ランダムに」「想定外の質問で鍛えたい」 | cold |
| 引数なしで起動 | 対話で確認 |
