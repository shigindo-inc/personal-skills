# Axis Tone — PO 軸 / 実装エンジニア軸別の強調・トーンダウンマトリクス

応募軸 × 業界 × 職種の組み合わせに応じて、`想定問答_カスタム.md` の `## カスタムの軸` に書き込む強調 / トーンダウンポイントの決定ロジック。

`23_応募先別カスタマイズ指針_PO軸.md` / `23_応募先別カスタマイズ指針_実装エンジニア軸.md` の縮約版。

## PO 軸

### 共通強調軸

- プロダクト全体の意思決定経験（副業 {{ side_projects.projects[0].name }} / {{ side_projects.projects[1].name }} での PO 経験）
- ステークホルダー折衝（オフショア窓口 / 副業のクライアント対応）
- 技術判断力（実装も自分でやる PO の希少性）
- 国際コミュニケーション（{{ qualifications.toeic }} 等）・{{ global_experience.offshore_collaboration_regions[] }} との協業）

### 共通トーンダウン

- 「コードを書きたい」「最新技術が好き」を前面に出さない（PO らしさが薄まる）
- 個人事業の詳述（商流・営業）
- 学歴ギャップ / 在籍年数の自己卑下

### 業界別

#### 自社開発 SaaS（B2B）— 最優先ターゲット

**強調**：
- カスタマー課題から逆算した意思決定経験（副業案件で「使われない機能を止めた」等の判断）
- 既存顧客の安定運用と新機能開発のバランス設計
- マルチテナント / エンタープライズ要件への配慮

**トーンダウン**：
- 純粋な技術的興味
- スタートアップ的な「とりあえず作ってみる」スタンス

#### AI スタートアップ / LLM 企業

**強調**：
- 生成 AI 業務での主担当経験（Azure OpenAI / OpenAI モデル選定）
- ハルシネーション / プロンプトインジェクション等のリスク認識
- AI コーディングエージェント（Claude Code）の業務適用判断力

**トーンダウン**：
- 「コードを書く時間」を強調しすぎる
- 受託・SES 出身であることの過剰説明

#### コンサルティング企業 / DX 支援

**強調**：
- 複数プロジェクトの並行マネジメント経験（副業 {{ side_projects.total_count }} 件）
- 提案ベースの技術選定経験
- クライアント技術判断への影響力

**トーンダウン**：
- 「1 つのプロダクトに腰を据えたい」スタンス
- 自社プロダクト志向の強調

#### スタートアップ（シード〜B）

**強調**：
- 不確実性下での意思決定経験
- 役割の流動性への適応（実装 + PO + PM 兼任）
- 自走力（社内研修なしでネットワーク → フルスタック転換）

**トーンダウン**：
- 大企業的なプロセス重視
- ワークライフバランスの強調

#### 外資系企業

**強調**：
- 英語 + 国際コミュニケーション（{{ qualifications.toeic }} 等）・{{ global_experience.overseas_stay_duration }} の海外滞在・オフショア窓口）
- グローバル本社との時差調整経験（{{ global_experience.offshore_collaboration_regions[] }} との協業）
- 数値 / KPI ベースの意思決定経験

**トーンダウン**：
- 国内に強く根ざした働き方への愛着
- 副業に過度な比重

### 職種別（PO 軸内）

#### PL / PM 候補

**強調**：副業での PL / PM 兼任、プロダクト全権オーナーシップ（{{ side_projects.projects[0].name }}）

**トーンダウン**：純粋な実装スキル

#### Tech Lead 候補

**強調**：実装 + チームリード（サブリーダー時代）、技術選定権限経験

**トーンダウン**：PO 寄りすぎる発言（プロダクト判断より技術判断を前面に）

#### AI エンジニア

**強調**：LLM 主担当・モデル選定・SDK 移行

**トーンダウン**：マネジメント志向

## 実装エンジニア軸

### 共通強調軸

- 自走力（社内研修なしでネットワーク → フルスタック転換、Python 独学）
- AI / LLM 実装力（OpenAI / Azure OpenAI / Anthropic の使い分け）
- オフショア窓口（Cross-functional コミュニケーション）
- 個人開発 / 副業（{{ side_projects.projects[0].name }} / {{ side_projects.projects[1].name }} / 例：クラウドソーシング、計 {{ side_projects.total_count }} 件）
- リーダーシップ（必要に応じて、サブリーダー・PL/PM 兼任）

### 共通トーンダウン

- 「マネジメント志向」「PO になりたい」を前面に出さない（実装軸が薄まる）
- 個人事業の商流詳述
- ラグビー副主将を最前面に出す（実装スキルから話が逸れる）

### 業界別

#### 自社開発 SaaS（B2B）

**強調**：フルスタック実装（Java / Python / TypeScript / Vue.js）、自走力、AI / LLM 実装

**トーンダウン**：受託案件特有の話、SES 階層構造の不満

#### AI スタートアップ / LLM 企業

**強調**：生成 AI 主担当（モデル選定・SDK 移行・新モデル追加）、AI コーディングエージェント業務利用

**トーンダウン**：純粋なバックエンド寄りの話

#### コンサルティング企業

**強調**：複数案件・複数スタック対応力、フルスタック

**トーンダウン**：1 つのプロダクトへのこだわり、自社プロダクト志向

#### スタートアップ

**強調**：自走力、副業での 0 → 1 実装、不確実性耐性

**トーンダウン**：プロセス重視

#### 外資系企業

**強調**：英語 + AI 実装、グローバル開発経験

**トーンダウン**：国内志向、副業比重

### 職種別（実装軸内）

#### フルスタックエンジニア

**自己 PR 順序**：自走力 → AI 実装 → オフショア窓口 → リーダーシップ

**トーンダウン**：マネジメント比重

#### AI エンジニア

**強調**：LLM / RAG / プロンプトエンジニアリング、モデル選定

**トーンダウン**：純粋なフロントエンドスキル

#### SRE / インフラ寄り

**強調**：Python 自動化（例：前職）時代）、IaC / CI/CD 構築意欲

**トーンダウン**：プロダクト判断・PO 系の経験
（ただし応募ターゲットとしては優先度低）

## 強調 3 件・トーンダウン 3 件の選定ルール

1. **業界別の強調軸 3 件**を基本セット
2. **職種別の強調を 1 件**追加（重複時は業界優先）
3. **共通強調軸から 1 件**追加（業界 / 職種で言及済みなら省略）
4. トーンダウンは **業界 + 共通から 1〜3 件**

## 出力例

```markdown
## カスタムの軸

- **応募軸**：実装エンジニア軸（AI エンジニア寄り）
- **強調ポイント**：
  - LLM 主担当経験（Azure OpenAI / OpenAI のモデル選定・SDK 移行・新モデル追加）
  - フルスタック実装力（Java / Python / TypeScript / Vue.js）
  - AI コーディングエージェント業務利用（Claude Code）
  - 自走力（社内研修なしのネットワーク → フルスタック転換）
- **トーンダウンするポイント**：
  - マネジメント志向の前面提示（PO 軸の話に寄りすぎない）
  - 個人事業の商流・販売手法
  - 受託・SES 出身であることの過剰説明
```
