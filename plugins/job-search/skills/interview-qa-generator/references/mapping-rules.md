## 当社特化質問（Q20〜）の生成テンプレ

`企業情報.md` の以下要素から、面接官が聞きたい論点を導出：

### 事業フェーズから

| フェーズ | 想定される面接官の質問 |
|---------|------------------|
| シード / アーリー | 「不確実性の高い環境での意思決定経験は？」「0 → 1 を経験した個人事業の具体的話を教えてほしい」 |
| ミドル | 「組織拡大期にスケールする仕組み作り経験は？」「他職種との連携で苦労した話」 |
| レイト / 上場後 | 「大規模組織での意思決定スピードへの慣れは？」「コンプライアンス / 監査対応経験」 |

### 技術スタックから

| スタック | 想定される面接官の質問 |
|---------|------------------|
| 弊社のメインスタック ⊃ ユーザーの経験 | 「即戦力としてのキャッチアップ期間は？」「具体的なコード経験を 1 つ」 |
| 弊社のメインスタック ⊄ ユーザーの経験 | 「未経験スタックへの学習計画は？」「過去の言語 / FW 乗り換え経験」 |
| AI / LLM が中核 | 「モデル選定の判断軸は？」「ハルシネーション・プロンプトインジェクション対策の知見」 |

### グリーン / レッドフラグから

| フラグ | 想定される面接官の質問 |
|------|----------------------|

## 要カスタムマーカーの判定

既存 Q に `> [このQ要カスタム：...]` マーカーを挿入する判定：

| Q | マーカー挿入条件 |
|---|--------------|
| Q2 志望動機 | **常に** 挿入（企業特化必須） |
| Q3 キャリアビジョン | 企業に Product Lead / IC キャリアパス事例の言及がある場合 |
| Q4 得意な技術領域 | 企業の技術スタックと自分の経験の overlap / gap が大きい場合 |
| Q6 AI / LLM 活用 | 企業の LLM 活用領域がマスタ B-4 と乖離する場合（例：自社モデル開発 vs API 利用） |
| Q9 プロダクト意思決定 | PO 軸 + 当該領域の B-9 質問が想定される場合 |
| Q13 希望年収 | 提示年収レンジと `compensation.desired_salary_range` の関係で回答を変える |
| Q14 副業希望 | 企業の副業ポリシーが「禁止」「条件付き」「自由」のどれかで回答を変える |

マーカー本文の書式：
```
> [このQ要カスタム：理由 = {一文の理由} / 参照 = {マスタの該当箇所 or 素材ファイル}]
```

例：
```
### Q13. 希望年収

> [このQ要カスタム：理由 = 提示レンジが `compensation.desired_salary_range` に近いため、マスタ B-7a の希望年収回答を参照 / 参照 = 24 B-7a]

> —
```

## マスタにフィードバックすべき新出パターンの扱い

本スキルが「マスタの 24 にこの Q & A が無いが、本企業向けに必要」と判断した場合：

1. 企業特化版に追加（Q20〜枠）
2. `## 想定される深掘り・逆襲質問` セクションにも記録
3. ユーザーに「マスタ 24 への追加候補：Q-XX 〜」と提示（実際の 24 への追加は本スキルでは行わない、`interview-retrospective` 経由 or 手動）
