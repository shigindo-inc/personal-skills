# Offer Evaluator

複数内定の比較 + 交渉準備 + 辞退連絡テンプレ生成を一手で行うスキル。

## 設定ファイル

> `${USER_PORTFOLIO_ROOT}` は `${CLAUDE_PLUGIN_ROOT}/config/personal-config.local.yaml` の `paths.portfolio_root` から取得。
> パイロット skill (offer-evaluator) で先行実装、他 skill は将来展開（ADR-0004 参照）。

## 構成と場所（絶対パス固定）

```
PORTFOLIO_ROOT   = ${USER_PORTFOLIO_ROOT}
COMPANIES_DIR    = ${PORTFOLIO_ROOT}/20_面接対策/companies
DASHBOARD        = ${PORTFOLIO_ROOT}/30_応募活動/36_応募ダッシュボード.md
CHECKLIST        = ${PORTFOLIO_ROOT}/30_応募活動/32_企業評価チェックリスト.md
WISH_PO          = ${PORTFOLIO_ROOT}/20_面接対策/22_転職希望条件_PO軸.md
WISH_ENG         = ${PORTFOLIO_ROOT}/20_面接対策/22_転職希望条件_実装エンジニア軸.md
```

更新対象：`${DASHBOARD}` セクション 3「内定比較テーブル」

## 起動時フロー

1. **personal-config の読み込み**（パイロット）：`${CLAUDE_PLUGIN_ROOT}/config/personal-config.local.yaml` を Read して以下を解決：
   - `paths.portfolio_root` → `${USER_PORTFOLIO_ROOT}` の実体パス（以降の `${COMPANIES_DIR}` 等が解決可能になる）
   - `person.name` → 辞退連絡テンプレ署名の `{{ person.name }}` を実値に展開
   - ファイル不在時：example をコピーして `local` を作成するようユーザーに促す
2. **比較対象企業の確定**：
   - 引数 or 対話で対象企業を 2〜3 件指定
   - 指定無しなら `${COMPANIES_DIR}/*/README.md` から現フェーズが `内定` / `内定承諾` / `内定承諾検討中` の企業を全列挙
3. **各企業の必要ファイル存在確認**：README + 企業情報 + 面接ログ最低 1 件があるか
4. **応募軸の確認**：各企業の応募軸（PO / 実装）を README から取得、軸が異なる場合は別軸として比較

## 基本ワークフロー

### Step 1：データ収集（並列 Read）

各内定企業について：
- `${COMPANIES_DIR}/{企業名}/README.md`：基本情報・評価スコア・志望動機要旨
- `${COMPANIES_DIR}/{企業名}/企業情報.md`：事業内容・技術スタック・カルチャー観察メモ
- `${COMPANIES_DIR}/{企業名}/面接ログ/*.md`：手応え・グリーン/レッドフラグの実観察

加えて：
- `${WISH_PO}` and / or `${WISH_ENG}`：希望条件の正本
- `${CHECKLIST}`：評価軸の定義

### Step 2：内定比較テーブル生成

`${DASHBOARD}` セクション 3 を以下の列で更新（既存テンプレを尊重）：

| 項目 | 企業 A | 企業 B | 企業 C |
|------|--------|--------|--------|
| 企業名 | {名} | {名} | {名} |
| フォルダ | リンク | リンク | リンク |
| 職種 | {応募ポジション} | ... | ... |
| 応募軸 | PO / 実装 | ... | ... |
| 年収（提示） | XXX 万円 | ... | ... |
| 必須条件クリア | X / 5 | ... | ... |
| 希望条件スコア | X / 21 | ... | ... |
| 裁量・オーナーシップ（A1〜A5） | X / 5 | ... | ... |
| プロダクト環境（B1〜B4） | X / 4 | ... | ... |
| 会社規模（30〜200 名） | ○ / ✕ | ... | ... |
| 副業可 | ○ / ✕ / 条件付 | ... | ... |
| リモート可 | ○ / ✕ / 週 N 日 | ... | ... |
| グリーンフラグ数 | N | ... | ... |
| レッドフラグ数 | N | ... | ... |
| 総合判断 | （推奨度 + 理由） | ... | ... |

### Step 3：希望条件との照合

`${WISH_PO}` / `${WISH_ENG}` から以下の希望条件を抽出して、各社のオファー内容と照合：

- 年収レンジ下限（{{ compensation.minimum_salary }} 万円、または {{ compensation.desired_salary_range.lower }}〜{{ compensation.desired_salary_range.upper }} 万円）
- 副業可否優先度
- リモート要件（フルリモート必須 / 週 N 日可）
- 業界 NG リスト
- 想定退職時期との整合

照合結果を「決め手」「懸念点」として各社にメモ。

### Step 4：総合推奨順の提示

優先度の決定軸（上から順）：
1. 必須条件クリア数（5/5 必須）
2. 裁量・意思決定関与（A1〜A5、PO 軸では特に重要）
3. 年収（希望レンジ下限以上）
4. レッドフラグ数（多い = 大幅減点）
5. 副業・リモート条件

推奨順を 1 位 〜 N 位で提示し、各順位の理由を 2〜3 行で。

### Step 5：交渉スクリプト下書き

以下の条件に該当する企業について **交渉スクリプト下書き** を生成：

| 条件 | 交渉対象 |
|------|--------|
| 提示年収が希望レンジ下限近く（差 50 万円以内） | 年収交渉 |
| 副業条件「要確認」or「条件付」 | 副業条件の明文化要求 |
| リモート週 1〜2 日（希望は週 3 日以上） | リモート条件交渉 |
| 入社時期が早すぎ（現職退職期間考慮されていない） | 入社時期の調整 |

スクリプト形式：
```
【XX社向け 年収交渉スクリプト案】

担当者へ：

ご提示いただいたオファー、誠にありがとうございます。
〜（謝意・前向きな姿勢の確認）〜

つきましては、年収について 1 点ご相談させてください。
〜（現職水準 + 副業収入 + 他社オファーの実態から、希望レンジ下限を示す）〜

具体的には XXX 万円〜YYY 万円のレンジでご検討いただけないでしょうか。
〜（自分が提供できる価値 = 強調軸 3 件）〜

ご検討のほど、よろしくお願いいたします。
```

数値根拠は `${WISH_*}` から引用、創作しない。

### Step 6：辞退連絡テンプレ

選定外（推奨順 下位）の企業向けに辞退連絡メールテンプレを生成：

```
【XX社向け 辞退連絡テンプレ】

件名：選考辞退のご連絡（{{ person.name }}）

XX社 採用ご担当者様

〜（謝意：選考機会・面接で得た学び）〜

熟慮の結果、別の企業様で選考を進めることに決定いたしました。
ご期待に応えられず、誠に申し訳ございません。

御社の今後のご発展を心よりお祈り申し上げます。

{{ person.name }}
```

エージェント経由応募の場合は、エージェントへの連絡も併記。

### Step 7：次アクション

- 1 位企業：交渉スクリプトの最終確認 → 連絡
- 2 位以下：辞退連絡（タイミング調整、1 位の承諾後）
- 各社の README の現フェーズを更新（内定承諾 / 辞退）
- `${DASHBOARD}` セクション 1 の該当行も更新

## 操作カタログ

| 操作 | トリガー語 | 動作 |
|------|----------|------|
| 全工程実行（複数内定の比較〜辞退テンプレ） | 「○○社と××社、内定比較して」 | Step 1〜7 |
| 内定比較テーブルのみ生成 | 「内定比較テーブル更新」 | Step 1〜2 |
| 交渉スクリプトのみ生成 | 「○○社の年収交渉スクリプト」 | Step 5 のみ |
| 辞退連絡テンプレのみ生成 | 「○○社の辞退連絡テンプレ」 | Step 6 のみ |

## ファイル編集ルール

| 対象 | 操作 |
|------|------|
| `${DASHBOARD}` セクション 3 | 該当列を上書き or 追記 |
| `${DASHBOARD}` セクション 1 | 該当行の現フェーズ更新（Step 7 のみ） |
| `${COMPANIES_DIR}/{企業名}/README.md` 現フェーズ | 内定承諾 / 辞退に更新（Step 7 のみ） |
| 交渉スクリプト / 辞退テンプレ | 出力のみ（ファイル保存はユーザー判断） |
| `${WISH_*}`, `${CHECKLIST}`, `企業情報.md`, `面接ログ/*` | 編集不可（読み込みのみ） |

## 引数（スラッシュコマンド呼出時）

`/offer-evaluator` の引数：

| 引数 | 動作 |
|------|------|
| なし | 内定企業を自動列挙して比較 |
| `<企業A> <企業B> [<企業C>]` | 指定企業のみ比較 |
| `--negotiate <企業名>` | 交渉スクリプトのみ生成 |
| `--decline <企業名>` | 辞退連絡テンプレのみ生成 |
| `--no-dashboard-update` | ダッシュボード編集なし（出力のみ） |

## 責務分離（やらないこと）

| やらないこと | 委譲先 |
|------------|--------|
| 単一オファーの評価 | `${CHECKLIST}` を直接適用、本 skill 不要 |
| 内定承諾後の入社準備（条件書確認等） | 範囲外 |
| 面接ログの新規作成 | `interview-retrospective` |
| 企業情報の追加収集 | `company-prep --info-only` |
| メール送信そのもの | 手動（テンプレ提供のみ） |
| 辞退理由の創作 | 簡潔な定型文を使う、嘘の理由は付けない |

## 禁止事項 / 終了時チェックリスト

- ❌ 数値根拠を創作しない（年収・副業比率・希望条件は `${WISH_*}` から引用）
- ❌ 「辞退理由」に他社の悪口や具体的な企業名を入れない
- ❌ 交渉スクリプトで「家族の事情」を年収根拠にしない（24 D-3 ルール）
- ❌ 推奨順を 1 つに絞る前に、各順位の理由を明示する
- ❌ 内定承諾 / 辞退の連絡を本 skill が直接送らない
- ✅ 終了時：`${DASHBOARD}` セクション 3 が全企業分埋まっているか
- ✅ 終了時：推奨順が 1 位 〜 N 位で提示され、各順位の理由が記述されているか
- ✅ 終了時：交渉対象企業に交渉スクリプト下書きが提示されているか
- ✅ 終了時：選定外企業に辞退連絡テンプレが提示されているか
- ✅ 終了時：次アクション（タイミング含む）が明示されているか
