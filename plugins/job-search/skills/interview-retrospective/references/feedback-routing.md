# Feedback Routing — 抽出内容 → 各ファイルへの振り分け規則

面接ログから抽出した情報を、各管理ファイルにどう反映するかのルール。**全て、ユーザー承認後に編集** が原則。

## 振り分けマトリクス

| 抽出元（ログの章） | 振り分け先ファイル | 振り分け先セクション | 編集内容 |
|----------------|----------------|----------------|---------|
| 聞かれた主な質問（新出） | `24_面接想定問答_逆質問.md` | 該当章（A / B / C） | 質問本文 + 自分の回答 + 深掘り想定 |
| 答えられなかった質問 | `{企業}/想定問答_カスタム.md` | 該当 Q | 要カスタムマーカー追加、補強ドラフト |
| 答えられなかった質問のうち、内面論点 | `47_行動原理・性格分析.md` | `## 追加で深掘りしたい質問` | 質問本文を箇条書き追加 |
| グリーンフラグ | `{企業}/企業情報.md` | `## カルチャー観察メモ` → `**グリーンフラグ**` | 観察したフラグを箇条書き追加 |
| レッドフラグ | `{企業}/企業情報.md` | `## カルチャー観察メモ` → `**レッドフラグ**` | 観察したフラグを箇条書き追加 |
| 評価スコア再計算（必須 / 希望） | `{企業}/README.md` | `## 評価スコア` | スコア更新（仮 → 確定） |
| 逆質問への回答 | `{企業}/逆質問.md` | `## メモ` | 質問・回答・温度感を箇条書き追加 |
| 現フェーズ進行 | `{企業}/README.md` | `## 現フェーズ` | 次段階へ更新、最終更新日を当日に |
| ダッシュボード行 | `36_応募ダッシュボード.md` | セクション 1 | 現フェーズ・次アクション・期日・評価スコア更新 |

## 詳細ルール

### 1. 新出質問の `24_面接想定問答_逆質問.md` への取り込み

**判定**：聞かれた質問が、`${MASTER_QA}` の A-1〜A-6 / B-1〜B-9 / C-1〜C-5 のいずれにも該当しない、もしくは類似だが文脈が異なる → **新出** 扱い

**振り分け先章の判定基準**：

| 質問の性質 | 振り分け先章 |
|----------|-----------|
| 自己紹介 / 転職理由 / 強み弱み / キャリア観 | A 章 |
| 経歴・職歴・学歴ギャップ・副業の深掘り | B 章 |
| 設計 / テスト / デバッグ / コードレビュー / 技術的判断 | C 章 |
| 逆質問への応答パターン | E 章（補完） |
| 上記に属さない、企業特化的すぎる質問 | 24 には追加せず、当該 `想定問答_カスタム.md` Q20〜のみ |

**追記フォーマット**：

```markdown
### {新章番号}. {質問本文}

> {自分の答えた内容 or 答えるべき内容のドラフト}

**深掘り想定**：

- Q：{追加で予想される深掘り質問}
  - A：{回答ドラフト}
```

**承認フロー**：

```
新出質問 X 件を `24_面接想定問答_逆質問.md` に追加候補として提示します：

1. [章] {質問本文}
2. [章] {質問本文}
...

承認するもの番号で（複数可、`all` で全部、`none` で全部不採用）：
```

承認された質問のみ追記する。

### 2. 答えられなかった質問の `想定問答_カスタム.md` への補強

該当 Q（既存の Q1〜Q19 or 新規 Q20〜）の見出し直下にマーカーを追加：

```markdown
### Q{N}. {質問}

> [このQ要再カスタム：理由 = {YYYY-MM-DD の面接で答えに詰まった} / 補強案 = {素材 4X / 自己分析 47 から引用}]

{既存の本文 or `—`}

**面接ログ参照**：`面接ログ/{YYYY-MM-DD_ラウンド}.md`
```

加えて、補強ドラフトが提示できる場合（マスタ 24 や素材 4X に該当があれば）、`想定問答_カスタム.md` 末尾の `## 当社特化質問` 章にドラフト追記。

### 3. 内面論点の `47_行動原理・性格分析.md` への取り込み

`## 追加で深掘りしたい質問` 章に箇条書き追加：

```markdown
- {質問本文}（出典：{YYYY-MM-DD} {企業名} {ラウンド}面接で答えに詰まる）
```

追加後、ユーザーに `deep-self-analysis` 起動を提案：

```
内面論点 N 件を `47_行動原理・性格分析.md` の「追加で深掘りしたい質問」章に追加しました。

これらを `deep-self-analysis` で 1 問ずつ深掘りすることをおすすめします。次のセッションで「deep-self-analysis」と呼んでください。
```

### 4. グリーン / レッドフラグの `企業情報.md` への追記

`## カルチャー観察メモ` セクション：

**Before**（テンプレ）:
```markdown
**グリーンフラグ**（`32_企業評価チェックリスト.md` 参照）：

- —
```

**After**（観察フラグを追記）:
```markdown
**グリーンフラグ**（`32_企業評価チェックリスト.md` 参照）：

- CTO との週次 1on1 文化あり（YYYY-MM-DD 一次面接で確認、面接官の発言：「うちは CTO が毎週時間取ってます」）
- AI コーディングエージェント（Cursor）を業務利用奨励（YYYY-MM-DD 一次面接で確認）
```

各箇条書きに **観察日と面接官の具体的発言** を併記（再現性のため）。

### 5. 評価スコア再計算

`README.md` の `## 評価スコア` 表を `${CHECKLIST}` に従って再計算：

| 項目 | 算出ロジック |
|------|----------|
| 必須条件クリア | 必須 1〜5 のうち、面接で OK 確認できたもの数 / 5 |
| 希望条件スコア | A1〜E3 のうち、面接で確認できたもの数 / 21 |
| 裁量・オーナーシップ（A1〜A5） | A1〜A5 のうち面接で確認できた数 / 5 |
| プロダクト環境（B1〜B4） | B1〜B4 のうち確認できた数 / 4 |
| 会社規模（30〜200 名） | 従業員数から判定（既に確定済の場合は維持） |
| グリーンフラグ数 | 観察された数 |
| レッドフラグ数 | 観察された数 |

「仮 X / 5（面談で確定）」だった項目を「X / 5」に更新（仮の文字を外す）。

### 6. 逆質問への回答ログ

`{企業}/逆質問.md` の `## メモ` セクションに追記：

```markdown
## メモ

### YYYY-MM-DD {ラウンド}

- Q: {聞いた質問}
  - A: {返ってきた回答の要旨}
  - 温度感: 即答 / 言い淀み / 紋切り型
  - 自分の判断への影響: プラス / マイナス / 変わらず
```

### 7. 現フェーズの進行ロジック

| 現在のフェーズ | 手応え ◎○ → 次フェーズ | 手応え △✕ → 次フェーズ |
|------------|----------------|----------------|
| 一次面接予定 | 一次面接済 | 一次面接済 |
| 一次面接済 | 二次面接予定（先方からの連絡待ち） | 不合格懸念（連絡待ち / 辞退検討） |
| 二次面接済 | 最終面接予定 | 不合格懸念 |
| 最終面接済 | 内定（連絡待ち） | 不合格懸念 |
| 内定 | 内定承諾 / 他社比較中 → `offer-evaluator` | — |

**注**：手応えだけで `不合格` 確定はしない。先方からの正式連絡を待つ。`不合格懸念` は内部メモとして README にコメント。

### 8. ダッシュボード行の更新

`${DASHBOARD}` セクション 1 の該当企業行：

| 列 | 更新ロジック |
|----|----------|
| 現フェーズ | 上記 7 の進行後の値 |
| 次アクション | 手応えに応じて：「先方連絡待ち」「次ラウンド準備」「辞退検討」 |
| 期日 | 次ラウンド日 or 連絡期限 |
| 評価スコア | 再計算後の値 |
| メモ | 直近の印象 1 行（既存メモを置き換え or 追記） |

## 編集の優先順位

複数ファイル更新時の順序（依存関係 + 影響範囲）：

1. ログファイル自体（記憶の保全が最優先）
2. `企業情報.md`（グリーン / レッドフラグ追記）
3. `README.md`（評価スコア再計算 + フェーズ進行）
4. `想定問答_カスタム.md`（補強マーカー）
5. `逆質問.md`（メモ追記）
6. `${DASHBOARD}`（最後にサマリ反映）
7. `${MASTER_QA}` / `${BEHAVIOR_DOC}`（個別承認後、最後に）

各ステップで「次のファイルを更新していい？」と確認しながら進める。一気に全部書かない。
