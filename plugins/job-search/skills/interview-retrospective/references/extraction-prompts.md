# Extraction Prompts — 振り返り対話の質問テンプレ

対話モードで、面接後のユーザーから情報を引き出す質問パターン。記憶が鮮明なうちに、面接官の固有名詞・数値・フレーズを原文で記録するのが目的。

## 基本原則

1. **1 問ずつ**。複数並べない
2. **「言われた言葉そのまま」を求める**：要約させない。「面接官は実際にどういう言葉で？」と聞く
3. **多肢選択を活用**：印象・手応えは選択肢を提示
4. **沈黙を許容**：「思い出せない」と言われたら次に進む
5. **時系列に縛らない**：印象に残った順で聞く

## フェーズ別質問テンプレ

### 0. 面接情報の確認（最初）

```
まず面接の基本情報を確認させてください。
- 日時：何月何日 何時から何時まで？
- 形式：オンライン or 対面？オンラインなら何ツール？
- 面接官：何人？それぞれの肩書きは？（人事 / 現場エンジニア / マネージャー / 役員）
- 全体の時間：何分？
```

### 1. 聞かれた質問の抽出

```
面接で印象に残った質問から、思い出せる順に教えてください。後で番号を整えるので、順序は気にしなくて大丈夫です。
```

ユーザーが 1 件答えるごとに：

```
それに対して、ご自身ではどう答えましたか？
```

質問数 5 件未満で止まった時のフックアップ：

```
- 自己紹介はどの粒度で求められましたか？
- 志望動機の深掘りはどこまでされましたか？
- 技術的な質問（コード / 設計 / 言語）はありましたか？
- 経歴の中で学歴ギャップや転職理由が話題になりましたか？
- 副業や個人開発について聞かれましたか？
- 年収 / 入社時期 / リモート可否は話に出ましたか？
```

### 2. うまく答えられたこと

```
面接で「これは準備が活きた」「自分でも納得感のある回答ができた」と感じた質問はありましたか？
```

掘り下げ：
```
それはなぜうまく答えられたと思いますか？事前準備の何が効いた？
```

### 3. 答えられなかったこと

```
逆に「これは答えに詰まった」「もっと準備しておけばよかった」と感じた質問は？
```

掘り下げ：
```
答えに詰まった瞬間、頭の中では何を考えていましたか？
- 「該当する経験がなかった」
- 「経験はあるが言語化できなかった」
- 「想定外の角度から聞かれた」
- 「自分の中で結論が出ていない論点だった」
最も近いのはどれですか？
```

最後の選択肢（自分の中で結論が出ていない論点）を選んだ場合 → `47_行動原理・性格分析.md` の「追加で深掘りしたい質問」章への追加候補としてフラグ立て。

### 4. 企業への印象 — グリーンフラグ

`32_企業評価チェックリスト.md` のグリーンフラグ 6〜7 項目を提示：

```
以下のグリーンフラグのうち、今日の面接で「実際に観察できた」「面接官の発言から確認できた」ものはありますか？該当するものを全部選んでください。

1. CTO / PO と週次 1on1 がある等の具体的な文化説明があった
2. 面接官が製品愛を持って話していた（数字より、ユーザーや課題を語る）
3. AI コーディングエージェントを積極活用・奨励している
4. 「こんな機能を自分で提案して実現した」という先輩エンジニアの事例があった
5. 評価基準が「成果」であり、時間管理ではない
6. 技術ブログ・OSS 貢献・登壇を会社として奨励している
7. 副業を「スキルアップの手段」として前向きに捉える文化がある

該当するもの番号で：
```

選択された項目について：
```
{選択された項目} について、面接官の具体的な発言を覚えていますか？
```

### 5. 企業への印象 — レッドフラグ

```
逆に、以下のレッドフラグで「該当しそう」と感じたものはありますか？

1. 「まず仕様書に従って実装してください」「技術選定は上が決めます」の発言
2. 事業の具体的な数値・成長戦略を一切話せなかった
3. 「AI ツールは社内規定で禁止」等の過剰管理
4. 開発プロセスがウォーターフォール一択
5. エンジニアが「技術的な選択に口を出せない」雰囲気
6. 離職率が高い、チームの継続性が不明
7. 副業について「詳細は入社後に確認」とぼかされた
8. 年収上限が明らかに低い
9. 「まず言われたことをやってください」等の発言

該当するもの番号で（なければ「なし」）：
```

### 6. 逆質問の結果

```
こちらから聞いた逆質問のうち、印象に残った回答を 2〜3 件教えてください。

- どの質問をした？
- どんな回答が返ってきた？（具体的な言葉で）
- それを聞いて自分の中で印象は変わった？（プラス / マイナス / 変わらず）
```

### 7. 総合手応え

```
今日の面接の総合手応えは：

◎ 良好（次ラウンドが期待できる、企業も良い印象）
○ 普通（次に進めるかは五分五分）
△ やや不安（手応えなし、もしくは企業側にレッドフラグ）
✕ 厳しい（次は無さそう、もしくは見送り検討）

どれですか？理由を 1〜2 文で。
```

### 8. 自己分析論点のフックアップ

ステップ 3 で「自分の中で結論が出ていない論点」が出た場合、または手応えが △ / ✕ で根拠が「自分の語り方」にある場合：

```
今日の面接で答えに詰まった論点 / 違和感が残った論点を、`47_行動原理・性格分析.md` の「追加で深掘りしたい質問」章に追加してもいいですか？追加した後、`deep-self-analysis` で 1 問ずつ深掘りすることをおすすめします。
```

### 9. 次アクション

```
今日の面接を受けて、次に取るべきアクションを 2〜3 件挙げるとすると？
- 想定問答のどこを補強する？
- 企業情報で追加調査すべき項目は？
- 次ラウンドへの準備（あれば）
```

## バッチモードの解析手順

ユーザーがメモを貼付した場合：

1. **箇条書きの分類**：聞かれた質問 / 答えた内容 / 印象 / 逆質問 のどれかに分類
2. **対話モードの質問 1〜9 のうち、メモから読み取れない項目だけを質問**
3. **特に「面接官の具体的発言」と「総合手応え」はメモから読み取れないことが多い → 追加で聞く**

## メモ化のコツ（ユーザーへの助言）

面接直後のメモは以下を意識すると後の振り返りが楽：

- 面接官の **印象に残ったフレーズをそのまま**（自分の解釈ではなく）
- 答えに詰まった瞬間の **頭の中の感覚**（不安・苛立ち・空白等）
- 逆質問への **回答の温度感**（即答 / 言い淀み / 紋切り型）
- 全体の **時間配分**（自己紹介に何分使われたか）

これらは事後に思い出しにくいため、面接終了 30 分以内の記録を推奨。
