# Classification — FB の 5 カテゴリ分類規則

ユーザー FB（面接後の課題感・複数ログ横断の気づき）を 5 カテゴリに分類するルール。主カテゴリ 1 件 + 副カテゴリ任意。

## 5 カテゴリ定義

### C1：回答内容（個別 Q の答え方）

**特徴**：特定の質問への答えそのものが浅い / ズレている / 説得力不足。`24_面接想定問答_逆質問.md` の A〜C 章の個別 Q が対象。

**典型 FB**：
- 「B-4 生成 AI の語りが浅い、Azure / OpenAI / Anthropic の使い分けまで踏み込めない」
- 「A-5 弱みの答えが弱すぎる、深掘りされると詰まる」
- 「C-2 テスト戦略の答えが暗記的で薄い」
- 「Q4 得意領域を聞かれて 3 つも挙げて散らかった」

**分類の判定**：FB が **特定の Q や章番号** に紐づく / 「答え方」「説得力」「具体例不足」を訴える → C1。

### C2：経歴の語り方（時系列・転換理由）

**特徴**：時系列の説明・職種転換の理由・学歴ギャップ・在籍年数等、自分のキャリアパスの語り方が苦しい / 曖昧。`24` B 章（経歴の深掘り） + `40_素材/4X_職歴_*.md` が対象。

**典型 FB**：
- 「学歴ギャップの説明が苦しい、面接官が納得していない顔をする」
- 「{{ career.transition_topics[0] }} の理由が曖昧、突かれると沈黙する」
- 「（例：前職） 4 年と現職 1 年のバランス説明が不自然」
- 「副業の比率を聞かれると本業へのコミット説明が後手」

**分類の判定**：FB が **時系列・期間・転換理由・職歴の年数** に紐づく → C2。

### C3：強み・弱み・自己 PR

**特徴**：強み 3 つ・弱み・自己 PR・副業の説明等、応募軸を問わず使う「自己アピール」のブレ。`21_自己PR_アピール材料.md` + `24` A-4/A-5 + `50_職務経歴書関連/generator/draft_*.yaml` の self_pr 章 + 副業関連素材が対象。

**典型 FB**：
- 「副業の説明がブレる、案件によって違う話をしている」
- 「強み 3 つの並びが面接ごとにバラバラ、応募軸に合わせ切れていない」
- 「自己 PR が長い、結局何を売っているか伝わらない」
- 「弱みが『自分で巻き取りすぎる』だけで弱い、もう 1 つ厚みが欲しい」

**分類の判定**：FB が **強み / 弱み / 自己 PR / 副業の見せ方** に紐づく → C3。

### C4：志望動機・カスタマイズ指針

**特徴**：志望動機が刺さらない / 業界別の押しが弱い / 軸別のカスタマイズが浅い。`23_応募先別カスタマイズ指針_*.md` + `34_志望動機サンプル集.md` + 各社 `想定問答_カスタム.md` Q2 が対象。

**典型 FB**：
- 「PO 軸の志望動機が刺さらない、業界別の押しが弱い」
- 「自社開発 SaaS 向けのテンプレが弱い、競合との差別化が言えていない」
- 「AI スタートアップ向けに『LLM 主担当経験』だけだと薄い」
- 「コンサル向けには『複数案件並行』だけでなく『提案力』も入れたい」

**分類の判定**：FB が **志望動機・業界別の押し・カスタマイズ指針** に紐づく → C4。

### C5：自己理解の浅さ

**特徴**：「動機を言語化できない」「同じ質問を複数社で答えられない」「自分でも結論が出ていない論点」。`47_行動原理・性格分析.md` + `46_自己分析.md` が対象。これは **編集より先に深掘りが必要**。

**典型 FB**：
- 「同じ質問を 3 社で答えられない、本当の動機が自分でも分からない」
- 「『5 年後どうなりたいか』を聞かれると毎回違うことを言っている」
- 「PO 志向と実装志向、本当はどちらか分からない」
- 「弱みを聞かれた時、模範回答は言えるが自分でも違和感がある」

**分類の判定**：FB が **「自分でも分からない」「言語化できない」「結論が出ていない」** を含む → C5。

## 分類のフロー

1. FB テキストを精読
2. キーワード判定：
   - 「Q4」「B-4」等の章番号 → C1
   - 「学歴ギャップ」「転職理由」「在籍」「{{ career.transition_topics[0] }}」 → C2
   - 「強み」「弱み」「自己 PR」「副業の説明」 → C3
   - 「志望動機」「業界」「PO 軸」「実装軸」「カスタマイズ」 → C4
   - 「分からない」「言語化」「結論」「本当の動機」 → C5
3. 複数該当時：**最も影響範囲が広いもの** を主、他を副
   - C5 が含まれれば C5 を主（深掘り優先のため）
   - C4 と C1 なら C4 主（カスタマイズ指針が個別 Q を規定するため）
   - C3 と C1 なら C3 主（自己 PR が個別 Q の素材になるため）
   - C2 と C1 なら C1 主（経歴の語り方は B 章の個別 Q 集合のため）

## 分類が確定しない場合

FB が抽象的すぎる（「うまくいかなかった」「手応えがない」等）→ 1 問深掘り：

```
もう少し具体化させてください。以下のどれに最も近いですか？

A. 特定の質問への答えが浅いと感じた
B. 自分のキャリアの語り方（時系列・転換理由）が伝わらなかった
C. 強み / 弱み / 自己 PR の見せ方がブレた
D. 志望動機が刺さっていない感覚
E. そもそも自分でも答えに自信がない論点があった

最も近いのは：（A / B / C / D / E）

差し支えなければ、具体的な質問や場面を 1 つ教えてください。
```

選択肢への回答を C1〜C5 に写像（A=C1, B=C2, C=C3, D=C4, E=C5）。

## `--from-logs` モードでの自動分類

`${LOGS_GLOB}` から「答えられなかった / 準備不足だった点」「レッドフラグ」「逆質問への回答」を Grep し、頻出パターンで自動分類：

| Grep パターン | 推定カテゴリ |
|--------------|----------|
| 「答えに詰まった」+ 特定 Q 番号 | C1 |
| 「学歴ギャップ」「転職理由」「{{ career.transition_topics[0] }}」 | C2 |
| 「副業」「自己 PR」「強み」「弱み」 | C3 |
| 「志望動機」「業界」 | C4 |
| 「自分でも分からない」「言語化できない」 | C5 |

3 件以上の面接ログで同一カテゴリが頻出 → 「複数社で同じ弱み」として優先扱い。
