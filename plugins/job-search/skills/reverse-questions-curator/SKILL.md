# Reverse Questions Curator

企業別 `逆質問.md` を、企業情報の空欄・チェックリスト未確認項目・汎用 E 群から自動キュレーションするスキル。

## 設定ファイル

> `${USER_PORTFOLIO_ROOT}` は `${CLAUDE_PLUGIN_ROOT}/config/personal-config.local.yaml` の `paths.portfolio_root` から取得。
> パイロット skill (offer-evaluator) で先行実装、他 skill は将来展開（ADR-0004 参照）。

## 構成と場所（絶対パス固定）

```
PORTFOLIO_ROOT   = ${USER_PORTFOLIO_ROOT}
COMPANIES_DIR    = ${PORTFOLIO_ROOT}/20_面接対策/companies
MASTER_QA        = ${PORTFOLIO_ROOT}/20_面接対策/24_面接想定問答_逆質問.md
CHECKLIST        = ${PORTFOLIO_ROOT}/30_応募活動/32_企業評価チェックリスト.md
CUSTOM_PO        = ${PORTFOLIO_ROOT}/20_面接対策/23_応募先別カスタマイズ指針_PO軸.md
CUSTOM_ENG       = ${PORTFOLIO_ROOT}/20_面接対策/23_応募先別カスタマイズ指針_実装エンジニア軸.md
```

更新対象：`${COMPANIES_DIR}/{企業名}/逆質問.md`

## 起動時フロー

1. **企業フォルダ存在確認**：無ければ `company-prep` 起動を提案して中止
2. **応募軸の確定**：
   - `${COMPANIES_DIR}/{企業名}/README.md` の応募軸欄を読む
   - `未確定` / `PO 軸 / 実装エンジニア軸`（両方の選択肢が並んでいる初期状態）なら、ユーザーに確認
3. **ラウンドの確定**：`--round` 引数 or ユーザー指定（`1次` / `2次` / `最終`）。未指定時は「全ラウンド分まとめて生成」
4. **既存 `逆質問.md` を読む**：すでにユーザーが手動追記した質問は保持（追記主体）

## 基本ワークフロー

1. **読み込み**：
   - `${COMPANIES_DIR}/{企業名}/企業情報.md`（特に `## 確認しておきたい点` 章と "—" / `要確認` の残る項目）
   - `${COMPANIES_DIR}/{企業名}/README.md`（業界・規模・応募軸を確定）
   - `${CHECKLIST}` の必須条件（5 項目）+ 希望条件（A1〜E3）
   - `${MASTER_QA}` の E-1〜E-6 + E-3a
   - `${CUSTOM_PO}` or `${CUSTOM_ENG}`（業界別 / 職種別の確認軸）
2. **抽出**：
   - **企業情報空欄からの質問化**：[references/unknown-detection.md](references/unknown-detection.md) のロジックに従い、`—` / `要確認` 項目を質問に変換
   - **チェックリスト未確認項目からの質問化**：必須条件 2（副業可）・4（年収）等、面接で必ず確認すべき項目を質問化
   - **応募軸別の核心質問**：`${MASTER_QA}` の E-3a（PO 軸時）/ 実装エンジニア軸の核心質問を抜粋
   - **業界 / 職種別の補完質問**：[references/question-bank.md](references/question-bank.md) から該当業界・職種の頻出質問を追加
3. **`逆質問.md` 更新**（追記主体、既存セクション尊重）：
   - `## カテゴリ別質問` → 雛形を保持
   - `## 企業情報から導出した固有質問` → 抽出した固有質問で `—` を置換、既存追記は保持
   - `## 必ず確認すべき（応募軸別の核心質問）` → 該当軸セクションを充実
   - `## ラウンド別の使い分け` → `--round` 指定時は当該ラウンド行を強化、未指定時は全ラウンド分を整える
4. **避ける逆質問の再掲**：`${MASTER_QA}` E-6 のマナー違反質問を `## メモ` 末尾に再掲（マナー確認）
5. **次アクション提示**：「面接実施後は `interview-retrospective` で逆質問への回答を `## メモ` に記録」と伝える

## 操作カタログ

| 操作 | トリガー語 | 動作 |
|------|----------|------|
| 全ラウンド分の逆質問初版生成 | 「○○社の逆質問作って」 | 全工程実行（1 次〜最終） |
| 特定ラウンド向け強化 | 「○○社の一次逆質問」 | `--round 1次` で 1 次向けカテゴリのみ充実 |
| 企業情報空欄の質問化のみ | 「○○社の確認事項を質問に」 | `## 企業情報から導出した固有質問` のみ更新 |
| マナーチェック | 「○○社の逆質問マナー確認」 | `## メモ` の避ける質問再掲のみ |

詳細：
- 業界 / 職種別 質問テンプレ → [references/question-bank.md](references/question-bank.md)
- `—` / `要確認` 検出 → 質問化 ロジック → [references/unknown-detection.md](references/unknown-detection.md)

## ファイル編集ルール

| 対象 | 操作 |
|------|------|
| `${COMPANIES_DIR}/{企業名}/逆質問.md` | 追記 + 既存 `—` 置換 |
| `${COMPANIES_DIR}/{企業名}/企業情報.md` | 編集不可（読み込みのみ） |
| `${MASTER_QA}`, `${CHECKLIST}`, `${CUSTOM_PO}`, `${CUSTOM_ENG}` | 編集不可（読み込みのみ） |

ユーザーが手動追記した質問（`—` 以外の行）は **削除しない**。順序入れ替えのみ提案、削除は確認後。

## 引数（スラッシュコマンド呼出時）

`/reverse-questions-curator` の引数：

| 引数 | 動作 |
|------|------|
| `<企業名>` | 全ラウンド分生成（応募軸は README から自動取得 or 確認） |
| `<企業名> --round 1次\|2次\|最終` | 指定ラウンドの質問を強化 |
| `<企業名> --axis po\|engineer` | 応募軸を明示（README 上書き無し） |
| `<企業名> --unknown-only` | 企業情報空欄の質問化のみ |
| `<企業名> --manners-only` | 避ける質問の再掲のみ |

## 責務分離（やらないこと）

| やらないこと | 委譲先 |
|------------|--------|
| 企業情報の初期収集（`企業情報.md` の埋め） | `company-prep` |
| 想定問答の企業特化カスタム | `interview-qa-generator` |
| 面接後の逆質問への回答ログ | `interview-retrospective`（`## メモ` への追記） |
| `${MASTER_QA}` の E 章自体の更新（新カテゴリ追加など） | 手動 / `interview-retrospective` の振り返り経由 |

## 禁止事項 / 終了時チェックリスト

- ❌ `${MASTER_QA}` / `${CHECKLIST}` / `${CUSTOM_*}` を編集しない
- ❌ ユーザーが手動追記した質問を削除しない
- ❌ E-6（避ける逆質問）の質問を `## カテゴリ別質問` に混ぜない
- ❌ 業界 / 職種に該当しない `question-bank.md` 全質問を機械的に貼り付けない（厳選 3〜5 件）
- ✅ 終了時：`## 企業情報から導出した固有質問` に `—` が残っていないか
- ✅ 終了時：`## 必ず確認すべき（応募軸別の核心質問）` に該当軸セクションが充実しているか
- ✅ 終了時：`## メモ` 末尾に「避ける質問」が再掲されているか
- ✅ 終了時：「面接後は `interview-retrospective` で回答ログ」が次アクションに記載されているか
