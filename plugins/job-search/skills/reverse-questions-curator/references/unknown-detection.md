# Unknown Detection — `企業情報.md` の "—" / `要確認` を逆質問に変換するロジック

`20_面接対策/companies/{企業名}/企業情報.md` を読み、未記入項目を逆質問に変換するルール。

## 検出パターン

未記入と判定する文字列：
- 行頭から始まる `> —`（引用形式のプレースホルダ）
- `- —`（箇条書きのプレースホルダ）
- `| — |` / `| 要確認 |`（表セル内のプレースホルダ）
- `要確認` を含む行
- `（あれば）` を末尾に持つ項目（任意項目だが、聞ければ聞きたい）

## 出力形式

抽出した質問は、`逆質問.md` の `## 企業情報から導出した固有質問` セクションに以下の形式で書き込む：

```markdown
## 企業情報から導出した固有質問

> `企業情報.md` の「確認しておきたい点」と合わせて、本企業ならではの質問を書く。

- {質問本文}（背景：企業情報.md の `## XX` が未確認のため）
- {質問本文}（背景：32 必須条件 X が未確認のため）
- ...
```

`（背景：...）` の括弧書きは、面接当日には削除する想定で、準備段階での出処トレース用。

## 重複排除

抽出した質問が、すでに `## カテゴリ別質問` や `## 必ず確認すべき` に類似で含まれている場合は重複追加しない。判定基準：質問文の主語（聞いている対象）が同一なら重複扱い。
