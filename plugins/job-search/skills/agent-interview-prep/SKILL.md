# Agent Interview Prep

転職エージェント面談（人材紹介会社の担当者との面談）向けに、エージェント別の自己紹介スクリプトとチェックリストを生成するスキル。

## 設定ファイル

> `${USER_PORTFOLIO_ROOT}` は `${CLAUDE_PLUGIN_ROOT}/config/personal-config.local.yaml` の `paths.portfolio_root` から取得。
> 経歴・副業・希望条件に近い具体値は同ファイルの `career.*`, `education.*`, `side_projects.*`, `mobility.*`, `compensation.*`, `global_experience.*` から取得し、公開用 SKILL.md には実値を書かない。

## 構成と場所（絶対パス固定）

```
PORTFOLIO_ROOT   = ${USER_PORTFOLIO_ROOT}
AGENTS_DIR       = ${PORTFOLIO_ROOT}/30_応募活動/agents
AGENTS_INDEX     = ${AGENTS_DIR}/_index.md
SELF_INTRO       = ${PORTFOLIO_ROOT}/20_面接対策/25_エージェント面談用_自己紹介.md
WISH_PO          = ${PORTFOLIO_ROOT}/20_面接対策/22_転職希望条件_PO軸.md
WISH_ENG         = ${PORTFOLIO_ROOT}/20_面接対策/22_転職希望条件_実装エンジニア軸.md
GLOBAL_STRATEGY  = ${PORTFOLIO_ROOT}/30_応募活動/37_外資転職_統合戦略.md
```

更新対象（オプション）：`${AGENTS_DIR}/{エージェント}.md` の「次アクション」「最終更新」セクション

## 起動時フロー

1. **対象エージェントの確定**：
   - 引数 or 対話でエージェント名を指定（ファイル名形式：`（例：外資系エージェント）`, `（例：国内大手エージェント）` 等）
   - `${AGENTS_DIR}/{エージェント}.md` の存在確認
   - 存在しない → `${AGENTS_DIR}/_TEMPLATE.md` から新規作成するか確認
2. **面談ラウンドの確定**：
   - `--round` 引数 or 対話で：`初回` / `応募先紹介後フォロー` / `内定後` のいずれか
3. **対象エージェント特性の把握**：`{エージェント}.md` の温度感・過去のやり取り・取り扱い案件傾向を読む

## 基本ワークフロー

### Step 1：データ読み込み（並列）

- `${AGENTS_DIR}/{エージェント}.md`（個別ファイル）
- `${AGENTS_INDEX}`（横断比較表）
- `${SELF_INTRO}`（汎用自己紹介）
- `${WISH_PO}`, `${WISH_ENG}`（両軸の希望条件）
- 外資エージェント（例：外資系エージェント） 等）なら `${GLOBAL_STRATEGY}`

### Step 2：エージェント特性に応じたカスタマイズ

#### トーン調整

| エージェント特性 | トーン |
|--------------|------|
| 国内大手（リクルート / （例：エンジニア専門エージェント）等） | フォーマル、実績数値を強調 |
| 外資・グローバル（en world / Robert Walters 等） | 英語混じり可、{{ qualifications.toeic }} + {{ global_experience.overseas_stay_duration }} の海外経験を前面 |
| スタートアップ系（YOUTRUST / Wantedly 等） | カジュアル、副業 / 個人開発を強調 |
| ハイクラス専門（ビズリーチ / Direct Recruiting） | 経営層視点、PO / Tech Lead 志向を明示 |

#### 強調軸の選定

エージェントの取り扱い案件傾向に応じて、PO 軸 / 実装エンジニア軸のどちらをメインに据えるか：
- 国内 SaaS が多い → PO 軸の Product Lead 志向を前面
- AI スタートアップが多い → 実装エンジニア軸の LLM 実装力を前面
- 外資が多い → 両軸提示 + 国際コミュニケーションを軸足

#### 希望条件の出し方

- **初回面談**：年収レンジ・副業・リモートは下限を明示、上限は流動的に
- **応募先紹介後フォロー**：紹介された企業の温度感を尋ねる、こちらの条件は再強調
- **内定後**：他社オファー有無を伝えるかは戦略次第（`${GLOBAL_STRATEGY}` の方針に従う）

### Step 3：自己紹介スクリプト生成

`${SELF_INTRO}` の汎用版を骨子に、Step 2 のカスタマイズを適用した **30 秒版 + 2 分版** を出力。

形式：
```
【{エージェント名} 向け 自己紹介スクリプト（{ラウンド}）】

## 30 秒版

> （冒頭フックを {エージェント特性} に合わせて調整した版）

## 2 分版

> （詳細版、強調軸を {応募軸} に寄せた版）

## トーンメモ

- {重要：このエージェントには XX を強調 / YY をトーンダウン}
```

### Step 4：当日伝える項目チェックリスト

エージェント面談で必ず話す項目を、`${WISH_*}` から抽出してチェックリスト化：

```
【{エージェント名} 面談 持参チェックリスト】

## 必須伝達項目

- [ ] 現職の状況（{{ career.current_job.period_start }} から在籍、退職想定時期）
- [ ] 年収レンジ：下限 {{ compensation.desired_salary_range.lower }} 万円 / 上限 {{ compensation.desired_salary_range.upper }} 万円
- [ ] 副業可否：必須 / 任意（理由：{{ side_projects.disclosure_policy }} に沿って説明）
- [ ] リモート要件：{{ mobility.remote_preference }}
- [ ] 通勤可能エリア：{{ mobility.preferred_locations[] }}（出張可否：{{ mobility.business_trip_ok }}）
- [ ] NG 業界：{${WISH_*} から抽出}
- [ ] 想定入社時期：YYYY 年 M 月

## 提示書類

- [ ] 職務経歴書（PO 軸 or 実装エンジニア軸：{選択理由}）
- [ ] 履歴書（最新版）
- [ ] スキルシート（要望時のみ）

## 確認したい項目（エージェントへ）

- [ ] 取り扱い案件の業界 / 規模傾向
- [ ] 紹介スピード感（推薦から面接設定まで）
- [ ] 面接対策サポートの有無
- [ ] 過去の同様プロファイル（PO/AI 寄り）の年収実績
- [ ] 並行紹介の上限（複数エージェント可否）
```

### Step 5：次アクション + 個別ファイル更新

1. `${AGENTS_DIR}/{エージェント}.md` の「次アクション」セクションに「{面談日} 面談実施 → 紹介企業フォロー」等を追記
2. 「最終更新」を当日 YYYY/MM/DD に
3. 面談後は `interview-retrospective` の簡易版（エージェント面談用ログ）を `${AGENTS_DIR}/{エージェント}.md` に手動追記することを提案

## 操作カタログ

| 操作 | トリガー語 | 動作 |
|------|----------|------|
| 全工程実行 | 「{エージェント} 面談準備」 | Step 1〜5 |
| 自己紹介スクリプトのみ | 「{エージェント} 自己紹介スクリプト」 | Step 1〜3 |
| チェックリストのみ | 「{エージェント} 持参チェックリスト」 | Step 1, 2, 4 |
| エージェントファイル更新のみ | 「{エージェント} ファイル更新」 | Step 5 のみ |

## ファイル編集ルール

| 対象 | 操作 |
|------|------|
| `${AGENTS_DIR}/{エージェント}.md` の次アクション・最終更新 | 該当箇所のみ更新（Step 5） |
| `${AGENTS_INDEX}` | 編集不可 |
| `${SELF_INTRO}`, `${WISH_*}`, `${GLOBAL_STRATEGY}` | 編集不可（読み込みのみ） |
| 出力（自己紹介スクリプト / チェックリスト） | 画面出力のみ。ファイル保存はユーザー判断 |

## 引数（スラッシュコマンド呼出時）

`/agent-interview-prep` の引数：

| 引数 | 動作 |
|------|------|
| `<エージェント名>` | 全工程実行（ラウンドは対話で確認） |
| `<エージェント名> --round 初回\|フォロー\|内定後` | ラウンド明示 |
| `<エージェント名> --axis po\|engineer\|both` | 応募軸明示 |
| `<エージェント名> --intro-only` | 自己紹介スクリプトのみ |
| `<エージェント名> --checklist-only` | チェックリストのみ |

## 責務分離（やらないこと）

| やらないこと | 委譲先 |
|------------|--------|
| エージェント経由応募の企業フォルダ作成 | `company-prep --agent {名}` |
| 応募先企業の面接準備 | `interview-prep` agent |
| 個別オファー比較 | `offer-evaluator` |
| 職務経歴書の更新 | `cv-builder` |
| 履歴書（10_提出書類/）の更新 | 手動 |
| 外資 LinkedIn 経由のスカウト返信 | `sns-helper` |
| 面談ログそのものの作成（エージェント面談） | 簡易版を `${AGENTS_DIR}/{エージェント}.md` に手動追記 |

## 禁止事項 / 終了時チェックリスト

- ❌ 他エージェントの企業名・条件を本エージェントに開示しない（情報統制）
- ❌ 年収交渉の数値を `${WISH_*}` の範囲外で創作しない
- ❌ 副業先（{{ side_projects.projects[0].name }} / {{ side_projects.projects[1].name }}）の詳細をエージェントに自分から開示しない
- ❌ エージェント面談記録を `interview-retrospective` の正式ログにしない（エージェント面談 ≠ 企業面接）
- ✅ 終了時：自己紹介スクリプト（30 秒 + 2 分）が生成されているか
- ✅ 終了時：当日チェックリストが必須伝達・提示書類・確認項目に分かれて出力されているか
- ✅ 終了時：`${AGENTS_DIR}/{エージェント}.md` の次アクション・最終更新が更新されているか
- ✅ 終了時：エージェント特性に応じたトーンメモが付記されているか
