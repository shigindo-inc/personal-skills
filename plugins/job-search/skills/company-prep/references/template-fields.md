# Template Fields — `_TEMPLATE/` プレースホルダの自動埋め可否マトリクス

`20_面接対策/companies/_TEMPLATE/` 配下の全プレースホルダ（`—` / `要確認` を入れる箇所）の自動埋め可否を定義する。

## 凡例

| レベル | 意味 | company-prep の対応 |
|--------|------|------------------|
| **高** | URL から自動抽出可能 | WebFetch + 構造化抽出 |
| **中** | テンプレ + 手動調整 | 候補提示のみ、ユーザー承認後に埋める |
| **低** | 面接後 / 手動入力 | `要確認` で残す |

## README.md

### `## 基本情報`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 企業名 | 高 | ユーザー指定 |
| 公式 URL | 高 | ユーザー指定 or WebFetch |
| 業界 / 事業ドメイン | 高 | 公式トップ・会社概要 |
| 規模（従業員数） | 高 | 会社概要・採用ページ |
| 拠点 | 高 | 会社概要 |
| エージェント経由 | 高 | ユーザー指定（`--agent`） |
| 応募軸 | 中 | ユーザー指定（`--axis`）、未指定なら `未確定` |
| 提示年収レンジ | 高 | JD ページ |
| 副業可否 | 中 | JD or 福利厚生ページに記載があれば抽出、無ければ `要確認` |
| リモート | 高 | JD ページ |

### `## 現フェーズ`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 現フェーズ値 | 高 | 初期は `情報収集中` 固定 |
| 最終更新 | 高 | 当日 YYYY/MM/DD |

### `## 評価スコア`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 必須条件クリア | 中 | 必須 1/3/5 のみ WebFetch から仮判定、`仮 X / 5（面談で確定）` |
| 希望条件スコア | 低 | 面談後 |
| 裁量・オーナーシップ A1〜A5 | 低 | 面接後 |
| プロダクト環境 B1〜B4 | 中 | B1（プロダクト課題明示）、B3（AI 中核）は HP から仮判定可 |
| 会社規模（30〜200 名） | 高 | 従業員数から自動判定 |
| グリーンフラグ数 | 低 | 面談後 |
| レッドフラグ数 | 低 | 面談後 |

### `## 志望動機要旨`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 志望動機要旨（3 行） | 中 | `23_応募先別カスタマイズ指針_{軸}.md` の業界別テンプレ骨子（応募軸確定時のみ） |

### `## 次アクション`

|------|--------|--------|

## 企業情報.md

### `## 事業内容`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 主力プロダクト / サービス | 高 | 公式トップ / サービスページ |
| 事業フェーズ | 中 | 資金調達情報・従業員数・売上から推定（候補提示） |
| 売上規模・収益モデル | 中 | IR・PR Times、上場企業のみ高精度 |
| ターゲット顧客 / 市場規模 | 中 | 公式・PR Times・市場調査記事 |

### `## 技術スタック`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 言語 / フロント / バック / インフラ / DB / AI / モバイル / 開発手法 | 高 | JD ページの「技術スタック」「使用言語」項目 / エンジニアブログ |
| 自分のスタックとの一致度 | 中 | JD 抽出後に自動判定（Java/Python/TypeScript/Vue.js/Flutter/OpenAI/Azure OpenAI/Firebase との overlap） |

### `## 組織`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 従業員数 / エンジニア数 | 高 | 会社概要・採用ページ |
| 組織構造 | 中 | 会社概要・採用ページ（CTO/VPoE の有無）|
| 意思決定プロセス | 低 | 面談・面接 |
| 役職構造 | 中 | 採用ページ（職位の言及） |

### `## 採用情報`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 募集ポジション URL | 高 | ユーザー指定 |
| ジョブディスクリプション要旨 | 高 | JD ページ全文を 3〜5 行に要約 |
| 選考フロー | 中 | JD ページ末尾の「選考フロー」セクションがあれば抽出 |
| 想定年収レンジ | 高 | JD ページの年収欄 |

### `## カルチャー観察メモ`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| グリーンフラグ | 中 | エンジニアブログ・登壇情報 から候補提示、最終確認は面談後 |
| レッドフラグ | 低 | 面談・面接 |
| 社員の発信 | 高 | エンジニアブログ・登壇情報の URL 列挙 |
| 評判（OpenWork / 転職会議） | 低 | 手動（要会員登録のため WebFetch 不可） |

### `## 競合・市場ポジション`

| 項目 | レベル | 抽出元 |
|------|--------|--------|
| 競合・市場ポジション | 中 | 市場調査記事・公式の競合言及から候補提示 |
