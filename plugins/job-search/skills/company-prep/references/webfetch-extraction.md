# WebFetch Extraction — URL → 各章節への抽出ルール

`--url` で受け取った URL を WebFetch し、`企業情報.md` / `README.md` の各項目に振り分ける規則。

## 抽出対象 URL の分類

| URL の種類 | 主な抽出先 |
|------------|----------|
| 公式トップ / 会社概要 | 事業内容、組織、規模、拠点 |
| 採用トップ / Careers | 文化、採用情報、福利厚生、副業 |
| 個別 JD（求人票） | 募集ポジション、JD 要旨、選考フロー、年収、技術スタック |
| エンジニアブログ / 技術ブログ | 技術スタック、グリーンフラグ候補、社員の発信 |
| PR Times / プレスリリース | 事業フェーズ、資金調達、売上 |
| Wantedly / Findy | 組織、カルチャー、技術 |

## WebFetch プロンプトのテンプレ

WebFetch 呼出時のプロンプトは「抽出したい項目を明示する」スタイルで統一する：

```
このページから以下を抽出して JSON で返してください。該当が無ければ "要確認" を入れてください。

- company_name: 会社名
- business_summary: 主力プロダクト・サービスの 1〜2 文要約
- business_phase: シード / アーリー / ミドル / レイト / 上場後 のいずれか
- revenue_model: 収益モデル（B2B SaaS / B2C / 受託 / etc）
- target_customer: ターゲット顧客
- tech_stack: { language, frontend, backend, infra, db, ai, mobile, process }
- employee_count: 従業員数（数値 or 範囲）
- engineer_count: エンジニア数（あれば）
- locations: 拠点リスト
- has_cto: bool, has_vpoe: bool, has_tl: bool
- jd_summary: 募集ポジションの 3〜5 行要約
- selection_flow: 選考フロー（あれば配列で）
- salary_range: 想定年収レンジ（例 "650-900 万円"）
- remote_policy: フルリモート / ハイブリッド / 出社 / 不明
- side_job_policy: ○ / ✕ / 要確認
- engineer_blog_urls: エンジニアブログ・登壇 URL 配列
```

## 必須条件 1/3/5 の仮判定ロジック

`32_企業評価チェックリスト.md` の必須条件のうち、URL から仮判定可能なもの：

| # | 項目 | 仮判定ロジック |
|---|------|------------|
| 1 | 多重下請け SES でないか | JD に「自社プロダクト」「自社サービス」「プロダクト開発」の明示 → `仮 OK` / 「お客様先常駐」「受託開発専業」の記載 → `仮 NG` / 不明 → `要確認` |
| 3 | 勤務地 / リモート条件 | `locations` に `{{ mobility.preferred_locations[] }}` のいずれかを含む → `仮 OK` / `remote_policy` が `{{ mobility.remote_preference }}` と整合 → `仮 OK` / それ以外 → `要確認` |
| 5 | レガシー技術のみでないか | `tech_stack` の `language` に "TypeScript" / "Python" / "Go" / "Rust" / "Kotlin" / "Swift" / "Flutter" 等のモダン言語が 1 つでも入る → `仮 OK` / "COBOL" / "VB6" 単独 → `仮 NG` / 不明 → `要確認` |

必須条件 2（副業可）と 4（年収 `{{ compensation.minimum_salary }}` 以上）は仮判定の精度が低いため、`要確認` で残し、面談で必ず聞く。

## 業界の判定

`business_summary` + `tech_stack` から業界を 1 つ選ぶ（複数該当時はメインを 1 つ）：

| 業界カテゴリ | 判定キーワード |
|------------|--------------|
| 自社開発 SaaS | 「SaaS」「業務支援」「クラウドサービス」 + B2B |
| AI スタートアップ / LLM 企業 | 「生成 AI」「LLM」「機械学習」を事業の中核に置く + AI/ML スタック充実 |
| コンサルティング企業 | 「コンサルティング」「DX 支援」+ プロジェクト型受託 |
| スタートアップ（シード〜B） | 従業員数 < 50 名 + 資金調達履歴あり |
| 外資系企業 | 本社所在地が日本国外 + 英語 JD |

この業界カテゴリは `23_応募先別カスタマイズ指針_{軸}.md` の該当セクション選択に使う。

## 抽出失敗時のフォールバック

- WebFetch がタイムアウト / 403 / robots.txt ブロック → 当該 URL を「アクセス不可」として記録し、該当項目を `要確認` で残す
- HTML が空 / JS レンダリング前提 → 同上
- ユーザーに「URL アクセス不可のため手動入力が必要」と簡潔に報告。複数 URL のうち 1 つでも成功すれば、成功分のみで初期埋めを進める

**禁止**：失敗時に類似企業の情報・LLM 内の事前知識で穴埋めしない。すべて `要確認` で残し、ユーザーが確認する。

## エンジニアブログ URL の収集

`engineer_blog_urls` は `企業情報.md` の `## カルチャー観察メモ` → `**社員の発信**` 節に列挙する。各 URL は箇条書きで `- {URL}（{記事タイトル or 概要}）` 形式。

3 つ以上見つかった場合は「グリーンフラグ候補：技術発信奨励文化」を `**グリーンフラグ**` 節に候補として追加（最終判定は面談後）。
